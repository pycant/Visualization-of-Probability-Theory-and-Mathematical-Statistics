## 背景实现
- 在 `chapter1.html` 添加全屏 `<canvas id="bg-noise">`，绝对定位置于最底层；在其上放置内容容器（hero 区、章节内容占位）。
- 在 `chapter1.js` 新增 `setupPerlinBackground()`：纯 JavaScript 实现 2D 柏林噪声，按低分辨率网格生成噪声并缩放到屏幕尺寸；通过 `requestAnimationFrame` 做平滑动画，时间维使用噪声的第三维或相位偏移实现“明暗缓动”。
- 色彩方案：噪声值映射到 HSL 深色系渐变（如 H≈210–270、S≈30%–60%、L≈6%–18%），叠加径向渐变与轻微模糊，形成层次丰富但不过度干扰的背景。
- 卡片与内容层：为主内容卡片应用半透明玻璃态背景（opacity 0.85），确保可读性，同时利用已有 `fx-holo`/`fx-ambient` 风格保持一致性。

## 标题栏设计
- 新增 hero 标题区，居中显示章节标题与副标题，标题文本使用 HSL 渐变背景裁切（`background-clip: text`）实现自然色相过渡（CSS keyframes 动画），并添加轻微阴影增强立体感。
- 响应式排版：标题字号用 `clamp()`（如 `clamp(1.8rem, 3.5vw, 3.2rem)`）与 Tailwind 工具类，使在移动与桌面端均保持良好占比；副标题与说明文字按断点调整间距与行高。
- 在 hero 区底部放置“快速跳转”按钮组，与导航 `NAV_CONFIG.sectionLinks` 保持一致，点击平滑滚动到各节锚点。

## JavaScript 结构
- `chapter1.js` 新增两个模块：
  - `setupPerlinBackground()`：创建/调整画布、生成噪声帧、颜色映射、动画循环与可见性管理（`visibilitychange`）。
  - `setupTitleBar()`：为标题渐变与阴影做轻量交互（如悬停增强）、为跳转按钮绑定平滑滚动。
- 保持现有 `DOMContentLoaded` 注册点，按顺序执行：背景 → 标题栏 → 已有骰子与条件概率交互；模块之间互不耦合，便于独立维护。

## 性能优化
- 低分辨率采样（如 320×180）+ 最近邻或双线性放大绘制到全屏，显著降低 CPU 开销。
- 降帧自适应：若单帧耗时超过阈值则启用帧跳过（例如 30→24 FPS）；在失焦/不可见时暂停动画。
- 使用 `OffscreenCanvas`（支持时）或主线程 Canvas 2D；避免大数组频繁分配，复用缓冲区。
- 颜色映射预计算查表（LUT），减少每像素三角/幂运算。

## 浏览器兼容
- 依赖：原生 Canvas 2D + CSS3 动画；检测失败时 fallback 到静态深色渐变背景（保留页面可用性）。
- 兼容现代主流浏览器（Chromium/Firefox/Safari）；移动端通过 DPR 适配与降帧保证流畅。
