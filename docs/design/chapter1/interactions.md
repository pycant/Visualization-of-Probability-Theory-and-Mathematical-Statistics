# 第1章交互与练习设计

## 统一交互原则
- 控件节奏：滑块/开关/按钮大小一致；悬停白色、激活青色高光。
- 重置与回到默认：每个 `VisCard` 都提供“重置”按钮，恢复初始状态。
- 可访问性：键盘可操作（tab 顺序、`focus-visible` 高亮）；为图形提供对比度与替代文本。
- 性能：动画与模拟采用节流/请求帧；长循环使用 Web Worker（如蒙特卡罗大样本）。

## 用户流程（示例）
- 进入章节页 → 浏览 Hero 简介 → 在 TOC 选择小节 → 进入 `VisCard` → 参数调整 → 查看图形与结论 → 进入该大节习题 → 得到反馈与解析。

## 练习模块（Exercises）
- 题型：
  - 单选/多选：基础概念与性质应用。
  - 判断：事件关系与独立性。
  - 填空：小型计算或公式补全。
- 交互：
  - 提交即时反馈（对/错/部分正确）。
  - 显示解析（文字+图示）。
  - 错题重做与“显示提示”。
- 评分与进度：可选本地存储记录完成度。

## 反馈与提示
- 图形区域下方提供提示区：当用户设置不合理参数时，给出引导文本与修正建议。
- 公式旁提供“展开推导”按钮：显示步骤动画或文字推导。
